# 📘 DArt-B Academic Archive 📘

본 저장소는 다트비(DArt-B) 학술제의 코드, 시각화 자료, 연구 주제 관련 산출물을 장기적으로 보관·관리하기 위한 공식 아카이브(Archive) 입니다.
기수별 프로젝트, 팀별 연구 결과, 분석 노트북, 발표 자료(PPT) 등을 체계적으로 누적하여, 앞으로 학술제가 이어질 때 참고 가능한 지식 기반(DB) 으로 활용하는 것을 목표로 합니다.


⸻

# 저장소 활용 목적
	•	학술제 프로젝트 자료의 영구 보관
	•	후속 기수에서 참고할 수 있는 데이터 분석 자산 구축
	•	연구 주제 선정 시 아이디어 참고용 레퍼런스 제공
	•	코드 및 분석 과정의 재현 가능성(Reproducibility) 확보

# 👥 사용자(참고자)를 위한 이용 안내 👥

본 저장소의 목적은 **참고(reference)** 입니다.
아래 내용을 반드시 숙지해 주세요.

🔸 1) 코드를 그대로 복사하여 사용하지 않습니다.

이 저장소는 학술제 자료를 참고하기 위한 곳이지,
과제·경진대회·학술제 프로젝트에 직접 제출하기 위한 템플릿이 아닙니다.

🔸 2) 동일한 주제를 모방하거나 전체 구조를 베끼지 않습니다.

이전 기수의 문제 정의, 모델 구조, 데이터 처리 방식 등을
동일하게 베끼는 것은 금지됩니다.

🔸 3) 저작권 및 윤리 사항 준수
	•	모든 프로젝트는 각 팀의 창작물입니다.
	•	코드·이미지·보고서·발표자료의 무단 사용은 금지됩니다.
	•	재사용이 필요할 경우 반드시 원저작자 동의를 받아야 합니다.

## 저작권 및 라이선스 안내
	•	본 저장소 내 자료의 저작권은 각 팀 또는 원저작자에게 있습니다.
	•	별도 라이선스가 명시되지 않는 한, 모든 자료는
참고 목적의 열람만 허용됩니다.
	•	외부 제출·배포·상업적 활용은 금지됩니다.

# 관리자를 위한 안내 (Import & 구조 유지)

학술제 자료는 다음과 같은 방식으로 저장소에 관리됩니다.

1) 프로젝트 자료 가져오기 (Repository Import)
	•	각 팀의 GitHub 레포지토리가 있는 경우
GitHub → Import Repository 기능을 사용하여
Academic-Conference Organization 내부로 가져옵니다.
	•	레포지토리 이름은 다음 규칙을 따릅니다:
YY-기수_팀명_프로젝트명
예: 25-2_SYNERGY_Order101

2) 파일 수동 업로드 (ZIP 또는 제출 자료)

외부 플랫폼에서 제출된 자료는 다음 경로로 업로드합니다.
~~~bash
/projects
   └── 25-2
       └── TeamA/
           ├── code/
           ├── visualization/
           └── slides/
~~~
3) 동일 팀명이 여러 번 등장할 경우

기수별로 폴더가 분리되므로 충돌 없음.
필요 시 README 내부에 프로젝트 설명을 추가합니다.
